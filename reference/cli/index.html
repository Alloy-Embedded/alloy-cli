    CLI reference#

    Every alloy <verb> rendered automatically from the Click tree at alloy_cli.main:cli. Adding a new command in commands/ surfaces the new verb on the next site build — no doc edits.

    For a one-page printable reference see the Cheatsheet. For long-form usage walkthroughs head to the User Guide.

    alloy#

    alloy — terminal-native developer surface for the Alloy embedded platform.

    The roadmap is sequenced under openspec/changes/ in this repository. Phase 1 (this proposal: bootstrap-alloy-cli) ships only the package skeleton. Subcommands (new, build, flash, debug, boards, add, mcp, doctor) land in subsequent proposals.

    Usage:

    alloy [OPTIONS] COMMAND [ARGS]...

    Options:

    Name Type Description Default
    --version boolean Show the version and exit. False
    --help, -h boolean Show this message and exit. False

    alloy erase#

    Erase the chip's flash through the lockfile-pinned probe-rs. Gated behind a TTY confirmation prompt; pass --auto / --yes to bypass in CI or non-interactive contexts. Pass --region to erase only part of the flash.

    Usage:

    alloy erase [OPTIONS]

    Options:

    Name Type Description Default
    --region text Region to erase. Repeat for multiple regions. Names (bootloader, appslot-a, …) resolve via the device IR; 0xBASE-0xEND ranges pass through unchanged. Default: chip-wide erase. Sentinel.UNSET
    --auto boolean Skip the confirmation prompt. Required in non-TTY contexts. False
    --yes boolean Alias for --auto (matches the common apt/dnf convention). False
    --probe text Explicit probe selector. Same shape as alloy reset --probe. None
    --project-dir directory Project root containing alloy.toml + .alloy/toolchain.lock. .
    --help, -h boolean Show this message and exit. False

    alloy monitor#

    Stream bytes from the target's debug UART (or RTT channel) to stdout. Press Ctrl+] to disconnect cleanly. Resolves the port from alloy.toml when the project declares a console UART; --port / --baud overrides.

    Usage:

    alloy monitor [OPTIONS]

    Options:

    Name Type Description Default
    --port file Serial device path (e.g. /dev/cu.usbmodem1234). Overrides autodetect. None
    --baud integer Baud rate. Overrides the project's [uart].debug config; falls back to 115200 when neither resolves. None
    --mode choice (raw | rtt) Stream source: raw UART bytes or probe-rs RTT channel. raw
    --ansi / --no-ansi boolean Pass through ANSI escape sequences. Default strips them so the log stays grep-friendly. False
    --probe text Probe selector (only meaningful in --mode rtt). None
    --project-dir directory Project root containing alloy.toml + .alloy/toolchain.lock. .
    --help, -h boolean Show this message and exit. False

    alloy new#

    Scaffold a new alloy-cli firmware project.

    Usage:

    alloy new [OPTIONS] NAME

    Options:

    Name Type Description Default
    --board text Board id (run alloy boards to list). Mutually exclusive with --device. None
    --device text Chip-only project: e.g. st/stm32g0/stm32g071rb. Mutually exclusive with --board. None
    --license choice (MIT | Apache-2.0 | BSD-3) License header for the generated LICENSE file. MIT
    --author text Copyright holder for the LICENSE template. Alloy User
    --git / --no-git boolean Initialise a git repo with a single 'alloy new' commit. True
    --force boolean Allow scaffolding into a non-empty directory. False
    --path directory Destination directory. Defaults to ./. None
    --from-example text Scaffold from a docs/EXAMPLES entry (e.g. 01-blinky, 02-uart-echo). Mutually exclusive with --board / --device. None
    --install-toolchain / --no-install-toolchain boolean Install the family's toolchain after scaffolding. Default in a TTY: Y (prompts unless --auto). Default elsewhere: N. None
    --auto boolean Skip every interactive confirmation. Combine with --install-toolchain to perform the install non-interactively. False
    --help, -h boolean Show this message and exit. False

    alloy reset#

    Reset the connected probe target. Default is a soft CPU reset; pass --hard to pulse nRST. Lockfile-aware: the probe-rs binary comes from .alloy/toolchain.lock when present.

    Usage:

    alloy reset [OPTIONS]

    Options:

    Name Type Description Default
    --soft / --hard boolean Reset method: --soft (CPU reset, default) or --hard (nRST line). True
    --halt-after-reset boolean Leave the core halted after reset so a debugger can attach. False
    --probe text Explicit probe selector (matches alloy flash --probe). Each field is optional — '0483' matches every ST-Link, '0483:374b' matches every ST-Link/V2-1, the full triple pinpoints one probe. None
    --project-dir directory Project root containing .alloy/toolchain.lock. .
    --help, -h boolean Show this message and exit. False

    alloy setup#

    Guided onboarding for a fresh machine: detect or scaffold a project, then install its toolchain through the shared orchestrator (the same path alloy new and alloy doctor --fix use).

    Usage:

    alloy setup [OPTIONS]

    Options:

    Name Type Description Default
    --board text Pre-pick a board (skips the picker step). None
    --family text Pre-pick a family; mutually exclusive with --board. None
    --auto boolean Suppress every interactive prompt with the default answer (Y on each install confirmation). False
    --no-tui boolean Force the line-based wizard even when STDIN is a TTY. (Wave 3: line-based is the only path; this flag is a forward-compatible no-op.) False
    --project-dir directory Project root. Defaults to the current directory. .
    --help, -h boolean Show this message and exit. False
